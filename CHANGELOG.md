# Change Log

## [0.4.0] — 2026-02-26

### Copilot / LM Tools

- Added `@transitland` chat participant with agentic tool-calling loop
- Added `#transitlandReadDmfr` — read a `.dmfr.json` file and return all feeds, operators, and URLs
- Added `#transitlandSearchFeeds` — search all workspace `.dmfr.json` files by URL, feed ID, or spec
- Added `#transitlandDmfrSchema` — return the DMFR JSON schema for field/type lookups
- Added `#transitlandGtfsSpec` — return the GTFS Schedule or GTFS-RT spec; supports per-file filtering
- Added `#transitlandCompareFeedSources` — compare two or more GTFS feed URLs and recommend the best `static_current`
- Added `#transitlandNtdSearch` — search the US National Transit Database by agency name or NTD ID; returns `us_ntd_id` and candidate GTFS URLs
- Added `#transitlandAgenciesInFeed` — list agencies in a GTFS feed for operator/`associated_feeds` setup
- Added `#transitlandSetField` — set or remove a dot-notation field in a DMFR feed or operator record
- Added `#transitlandAddFeed` — add a new feed record to a `.dmfr.json` file with duplicate URL detection
- Added `#transitlandAddOperator` — add an operator record (nested or top-level) with correct `associated_feeds` wiring
- Added `#transitlandDmfrFormat` — lint and normalize a `.dmfr.json` file via the transitland CLI
- Added `instructions/dmfr-assistant.instructions.md` as a Copilot chat instructions file (active when editing `.dmfr.json`)
- Enriched `#transitlandFeedInfo` output: tags, license, authorization, feed state, recent version history, all URL fields

### MCP Server

- Added `transitland_ntd_search`, `transitland_agencies_in_feed`, `transitland_set_field`, `transitland_add_feed`, `transitland_add_operator` tools to the standalone MCP server

### Other

- Renamed extension to "Transitland" (was "Transitland Atlas DMFR")
- Command IDs changed: `transitland-atlas-vscode-extension.*` → `transitland.*`
- Settings changed: `transitlandAtlas.*` → `transitland.*` (added `transitland.cliPath`)
- Switched build tooling to esbuild (multiple entry points: extension + MCP server)
- Extracted `TransitlandCLI` wrapper class with binary auto-detection
- Added VS Code Output Channel "Transitland" for CLI output
- Bumped minimum VS Code engine to 1.105.0
- Removed redundant `activationEvents` (auto-generated by VS Code from `contributes` declarations)
